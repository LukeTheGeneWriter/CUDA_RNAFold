Detailed Documentation
----------------------

.. container:: brief-description

    This module provides general functions that allow for an easy control of constrained secondary structure prediction and evaluation.

.. container:: detailed-description

    Secondary Structure constraints can be subdivided into two groups:
    
    * :ref:`Hard constraints <doxid-group__hard__constraints>` , and
    
    * :ref:`Soft constraints <doxid-group__soft__constraints>` .
    
    While Hard-Constraints directly influence the production rules used in the folding recursions by allowing, disallowing, or enforcing certain decomposition steps, Soft-constraints on the other hand are used to change position specific contributions in the recursions by adding bonuses/penalties in form of pseudo free energies to certain loop configurations.
    
    Secondary structure constraints are always applied at decomposition level, i.e. in each step of the recursive structure decomposition, for instance during MFE prediction. Below is a visualization of the decomposition scheme
    
    .. image:: recursions.svg
    
    
    
    .. image:: recursions
    
    For :ref:`Hard constraints <doxid-group__hard__constraints>` the following option flags may be used to constrain the pairing behavior of single, or pairs of nucleotides:
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_EXT_LOOP <doxid-group__hard__constraints_1ga9418eda62a5dec070896702c279d2548>` - Hard constraints flag, base pair in the exterior loop.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_HP_LOOP <doxid-group__hard__constraints_1ga79203702b197b6b9d3b78eed40663eb1>` - Hard constraints flag, base pair encloses hairpin loop.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_INT_LOOP <doxid-group__hard__constraints_1ga21feeab3a9e5fa5a9e3d9ac0fcf5994f>` - Hard constraints flag, base pair encloses an interior loop.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_INT_LOOP_ENC <doxid-group__hard__constraints_1ga0536288e04ff6332ecdc23ca4705402b>` - Hard constraints flag, base pair encloses a multi branch loop.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_MB_LOOP <doxid-group__hard__constraints_1ga456ecd2ff00056bb64da8dd4f61bbfc5>` - Hard constraints flag, base pair is enclosed in an interior loop.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_MB_LOOP_ENC <doxid-group__hard__constraints_1ga02a3d703ddbcfce393e4bbfcb9db7077>` - Hard constraints flag, base pair is enclosed in a multi branch loop.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_ENFORCE <doxid-constraints__hard_8h_1a1aa55f2c6347e670e003b1a765632dad>` - Hard constraint flag to indicate enforcement of constraints.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_NO_REMOVE <doxid-constraints__hard_8h_1a9fcac36535850ff612c7e6b1305304a1>` - Hard constraint flag to indicate not to remove base pairs that conflict with a given constraint.
    
    * :ref:`VRNA_CONSTRAINT_CONTEXT_ALL_LOOPS <doxid-group__hard__constraints_1ga886d9127c49bb982a4b67cd7581e8a5a>` - Constraint context flag indicating any loop context.
    
    However, for :ref:`Soft constraints <doxid-group__soft__constraints>` we do not allow for simple loop type dependent constraining. But soft constraints are equipped with generic constraint support. This enables the user to pass arbitrary callback functions that return auxiliary energy contributions for evaluation the evaluation of any decomposition.
    
    The callback will then always be notified about the type of decomposition that is happening, and the corresponding delimiting sequence positions. The following decomposition steps are distinguished, and should be captured by the user's implementation of the callback:
    
    * :ref:`VRNA_DECOMP_PAIR_HP <doxid-group__constraints_1ga8bd41ebc8039378d242e4e8c273716a5>` - Flag passed to generic softt constraints callback to indicate hairpin loop decomposition step.
    
    * :ref:`VRNA_DECOMP_PAIR_IL <doxid-group__constraints_1gaeab04f34d7730cff2d651d782f95d857>` - Indicator for interior loop decomposition step.
    
    * :ref:`VRNA_DECOMP_PAIR_ML <doxid-group__constraints_1gaa15b1185673f0b9e900c4748d45f388f>` - Indicator for multibranch loop decomposition step.
    
    * :ref:`VRNA_DECOMP_ML_ML_ML <doxid-group__constraints_1ga735517266f2e35e1374b8f1ea77ef23e>` - Indicator for decomposition of multibranch loop part.
    
    * :ref:`VRNA_DECOMP_ML_STEM <doxid-group__constraints_1ga4a23054c75d8efc785de50e3ea87602f>` - Indicator for decomposition of multibranch loop part.
    
    * :ref:`VRNA_DECOMP_ML_ML <doxid-group__constraints_1ga7f4cb9ff7a33e67f0539bd39e7b19a78>` - Indicator for decomposition of multibranch loop part.
    
    * :ref:`VRNA_DECOMP_ML_UP <doxid-group__constraints_1gae6478dda14e50e2f2cb9ef333a29256e>` - Indicator for decomposition of multibranch loop part.
    
    * :ref:`VRNA_DECOMP_ML_ML_STEM <doxid-group__constraints_1ga63d8ceb8c96ae3b463e529e28cc0fe98>` - Indicator for decomposition of multibranch loop part.
    
    * :ref:`VRNA_DECOMP_ML_COAXIAL <doxid-group__constraints_1ga4fe48d575830b16c208e280e01ab1497>` - Indicator for decomposition of multibranch loop part.
    
    * :ref:`VRNA_DECOMP_EXT_EXT <doxid-group__constraints_1ga437adf5115c1999304eff26b41e4c9b6>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_UP <doxid-group__constraints_1gaff1ddaffe86d984623910b40cc8a8717>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_STEM <doxid-group__constraints_1gae44b5ace0d9b4a29088069ecb4cec441>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_EXT_EXT <doxid-group__constraints_1ga803bd818b3f4b2b0a4a5cfa2f7dc2045>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_STEM_EXT <doxid-group__constraints_1gabb09c5b78b75a44502fc77b950125c1e>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_STEM_OUTSIDE <doxid-group__constraints_1gae7554cd3ff089360c02e4920229e221c>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_EXT_STEM <doxid-group__constraints_1ga06efd054c9271438f6d82d4559d9e69f>` - Indicator for decomposition of exterior loop part.
    
    * :ref:`VRNA_DECOMP_EXT_EXT_STEM1 <doxid-group__constraints_1ga2e75d7a77118735b32f25422d9686719>` - Indicator for decomposition of exterior loop part.
    
    Simplified interfaces to the soft constraints framework can be obtained by the implementations in the submodules
    
    * :ref:`SHAPE reactivity data <doxid-group___s_h_a_p_e__reactivities>` and
    
    * ligands.
    
    An implementation that generates soft constraints for unpaired nucleotides by minimizing the discrepancy between their predicted and expected pairing probability is available in submodule :ref:`Generate soft constraints from data <doxid-group__perturbation>` .

        void vrna_constraints_add (
	    :ref:`vrna_fold_compound_t<doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`* vc,
	    const char* constraint,
	    unsigned int options
	    )

    .. container:: brief-description

        Add constraints to a :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` data structure.

    .. container:: detailed-description

        Use this function to add/update the hard/soft constraints The function allows for passing a string 'constraint' that can either be a filename that points to a constraints definition file or it may be a pseudo dot-bracket notation indicating hard constraints. For the latter, the user has to pass the :ref:`VRNA_CONSTRAINT_DB <doxid-group__hard__constraints_1ga4bfc2f15c4f261c62a11af9d2aa80c90>` option. Also, the user has to specify, which characters are allowed to be interpreted as constraints by passing the corresponding options via the third parameter.
        
        The following is an example for adding hard constraints given in pseudo dot-bracket notation. Here, ``vc`` is the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` object, ``structure`` is a char array with the hard constraint in dot-bracket notation, and ``enforceConstraints`` is a flag indicating whether or not constraints for base pairs should be enforced instead of just doing a removal of base pair that conflict with the constraint.
        
        .. ref-code-block:: cpp
        
            unsigned int constraint_options = :ref:`VRNA_CONSTRAINT_DB_DEFAULT <doxid-group__hard__constraints_1ga1c3864bdc92147a4d93de2b1b4356177>`;
        
            if (enforceConstraints)
              constraint_options |= :ref:`VRNA_CONSTRAINT_DB_ENFORCE_BP <doxid-group__hard__constraints_1ga29ebe940110d60ab798fdacbcdbbfb7d>`;
        
            if (canonicalBPonly)
              constraint_options |= :ref:`VRNA_CONSTRAINT_DB_CANONICAL_BP <doxid-constraints__hard_8h_1a10fadeec2d291a851fe2197051cf842c>`;
        
            :ref:`vrna_constraints_add <doxid-group__constraints_1ga35a401f680969a556858a8dd5f1d07cc>`(fc, (const char *)cstruc, constraint_options);
        
        In constrat to the above, constraints may also be read from file:
        
        .. ref-code-block:: cpp
        
            :ref:`vrna_constraints_add <doxid-group__constraints_1ga35a401f680969a556858a8dd5f1d07cc>`(fc, constraints_file, :ref:`VRNA_OPTION_DEFAULT <doxid-group__fold__compound_1gacea5b7ee6181c485f36e2afa0e9089e4>`);
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *

                - The fold compound
        
            *
                - constraint
        
                - A string with either the filename of the constraint definitions or a pseudo dot-bracket notation of the hard constraint. May be NULL.
        
            *
                - options
        
                - The option flags

        void vrna_message_constraint_options (unsigned int option)

    .. container:: brief-description

        Print a help message for pseudo dot-bracket structure constraint characters to stdout. (constraint support is specified by option parameter)

    .. container:: detailed-description

        Currently available options are:
        
        :ref:`VRNA_CONSTRAINT_DB_PIPE <doxid-group__hard__constraints_1ga13053547a2de5532b64b64d35e097ae1>` (paired with another base)
        
        :ref:`VRNA_CONSTRAINT_DB_DOT <doxid-group__hard__constraints_1ga369bea82eae75fbe626f409fa425747e>` (no constraint at all)
        
        :ref:`VRNA_CONSTRAINT_DB_X <doxid-group__hard__constraints_1ga7283bbe0f8954f7b030ecc3f2d1932b2>` (base must not pair)
        
        :ref:`VRNA_CONSTRAINT_DB_ANG_BRACK <doxid-constraints__hard_8h_1ad54c1315a47d55653dcaa5de6e544b77>` (paired downstream/upstream)
        
        :ref:`VRNA_CONSTRAINT_DB_RND_BRACK <doxid-group__hard__constraints_1gac17b034852c914bc5879954c65d7e74b>` (base i pairs base j)
        
        pass a collection of options as one value like this:
        
        .. code-block:: none
        
            vrna_message_constraints(option_1 | option_2 | option_n)
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - option
        
                - Option switch that tells which constraint help will be printed
