Detailed Documentation
----------------------

.. container:: brief-description

    All relevant functions to retrieve and copy pre-calculated energy parameter sets as well as reading/writing the energy parameter set from/to file(s).

.. container:: detailed-description

    This module covers all relevant functions for pre-calculation of the energy parameters necessary for the folding routines provided by RNAlib. Furthermore, the energy parameter set in the RNAlib can be easily exchanged by a user-defined one. It is also possible to write the current energy parameter set into a text file.

    .. container:: brief-description

        Get a data structure containing prescaled free energy parameters already transformed to Boltzmann factors.

    .. container:: detailed-description

        This function returns a data structure that contains all necessary precomputed energy contributions for each type of loop.
        
        In contrast to :ref:`vrna_params() <doxid-group__energy__parameters_1gad0e3e7e74bdc50d1709d40c92993185e>` , the free energies within this data structure are stored as their Boltzmann factors, i.e.
        
        :math:` exp(-E / kT) `
        
        where :math:` E ` is the free energy.
        
        If a NULL pointer is passed for the model details parameter, the default model parameters are stored within the requested :ref:`vrna_exp_param_t <doxid-group__energy__parameters_1ga01d8b92fe734df8d79a6169482c7d8d8>` structure.
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - md
        
                - A pointer to the model details to store inside the structure (Maybe NULL)
        
        
        
        .. rubric:: Returns:
        
        A pointer to the memory location where the requested parameters are stored
        
        
        
        .. admonition:: See also:
        
            :ref:`vrna_md_t <doxid-group__model__details_1ga1f8a10e12a0a1915f2a4eff0b28ea17c>` , :ref:`vrna_md_set_default() <doxid-group__model__details_1ga8ac6ff84936282436f822644bf841f66>` , :ref:`vrna_params() <doxid-group__energy__parameters_1gad0e3e7e74bdc50d1709d40c92993185e>` , vrna_rescale_pf_params()

        void vrna_exp_params_rescale (
	    :ref:`vrna_fold_compound_t<doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`* vc,
	    double* mfe
	    )

    .. container:: brief-description

        Rescale Boltzmann factors for partition function computations.

    .. container:: detailed-description

        This function may be used to (automatically) rescale the Boltzmann factors used in partition function computations. Since partition functions over subsequences can easily become extremely large, the RNAlib internally rescales them to avoid numerical over- and/or underflow. Therefore, a proper scaling factor :math:`s` needs to be chosen that in turn is then used to normalize the corresponding partition functions :math:`\hat{q}[i,j] = q[i,j] / s^{(j-i+1)}` .
        
        This function provides two ways to automatically adjust the scaling factor.
        
        #. Automatic guess
        
        #. Automatic adjustment according to MFE
        
        Passing ``NULL`` as second parameter activates the *automatic guess mode* . Here, the scaling factor is recomputed according to a mean free energy of ``184.3*length`` cal for random sequences. On the other hand, if the MFE for a sequence is known, it can be used to recompute a more robust scaling factor, since it represents the lowest free energy of the entire ensemble of structures, i.e. the highest Boltzmann factor. To activate this second mode of *automatic adjustment according to MFE* , a pointer to the MFE value needs to be passed as second argument. This value is then taken to compute the scaling factor as :math:` s = exp((sfact * MFE) / kT / length )` , where sfact is an additional scaling weight located in the vrna_md_t data structure of ``exp_params`` in ``vc`` .
        
        The computed scaling factor :math:`s` will be stored as ``pf_scale`` attribute of the ``exp_params`` data structure in ``vc`` .
        
        SWIG Wrapper Notes This function is attached to :ref:`vrna_fc_s <doxid-structvrna__fc__s>` objects as overloaded **exp_params_rescale()** method.
        
        When no parameter is passed to this method, the resulting action is the same as passing *NULL* as second parameter to :ref:`vrna_exp_params_rescale() <doxid-group__energy__parameters_1gad607bc3a5b5da16400e2ca4ea5560233>` , i.e. default scaling of the partition function. Passing an energy in kcal/mol, e.g. as retrieved by a previous call to the *mfe()* method, instructs all subsequent calls to scale the partition function accordingly.
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - vc
        
                - The fold compound data structure
        
            *
                - mfe
        
                - A pointer to the MFE (in kcal/mol) or NULL
        
        
        
        .. note::
        
            This recomputation only takes place if the ``pf_scale`` attribute of the ``exp_params`` data structure contained in ``vc`` has a value below ``1.0`` .
        
        
        
        .. admonition:: See also:
        
            :ref:`vrna_exp_params_subst() <doxid-group__energy__parameters_1ga8e7ac4fab3b0cc03afbc134eaafb3525>` , :ref:`vrna_md_t <doxid-group__model__details_1ga1f8a10e12a0a1915f2a4eff0b28ea17c>` , :ref:`vrna_exp_param_t <doxid-group__energy__parameters_1ga01d8b92fe734df8d79a6169482c7d8d8>` , :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`
