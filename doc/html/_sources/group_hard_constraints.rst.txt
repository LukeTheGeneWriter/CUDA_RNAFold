        typedef unsigned char () vrna_callback_hc_evaluate (
	    int i,
	    int j,
	    int k,
	    int l,
	    unsigned char d,
	    void *data
	    )

    .. container:: brief-description

        Callback to evaluate whether or not a particular decomposition step is contributing to the solution space.

    .. container:: detailed-description

        This is the prototype for callback functions used by the folding recursions to evaluate generic hard constraints. The first four parameters passed indicate the delimiting nucleotide positions of the decomposition, and the parameter ``denotes`` the decomposition step. The last parameter ``data`` is the auxiliary data structure associated to the hard constraints via :ref:`vrna_hc_add_data() <doxid-constraints__hard_8h_1a128920e0af52e4196a9d59fa13336c7c>` , or NULL if no auxiliary data was added.
        
        Notes on Callback Functions This callback enables one to over-rule default hard constraints in secondary structure decompositions.
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - i
        
                - Left (5') delimiter position of substructure
        
            *
                - j
        
                - Right (3') delimiter position of substructure
        
            *
                - k
        
                - Left delimiter of decomposition
        
            *
                - l
        
                - Right delimiter of decomposition
        
            *
                - d
        
                - Decomposition step indicator
        
            *
                - data
        
                - Auxiliary data
        
        
        
        .. rubric:: Returns:
        
        A non-zero value if the decomposition is valid, 0 otherwise
        
        
        
        .. admonition:: See also:
        
            :ref:`VRNA_DECOMP_PAIR_HP <doxid-group__constraints_1ga8bd41ebc8039378d242e4e8c273716a5>` , :ref:`VRNA_DECOMP_PAIR_IL <doxid-group__constraints_1gaeab04f34d7730cff2d651d782f95d857>` , :ref:`VRNA_DECOMP_PAIR_ML <doxid-group__constraints_1gaa15b1185673f0b9e900c4748d45f388f>` , :ref:`VRNA_DECOMP_ML_ML_ML <doxid-group__constraints_1ga735517266f2e35e1374b8f1ea77ef23e>` , :ref:`VRNA_DECOMP_ML_STEM <doxid-group__constraints_1ga4a23054c75d8efc785de50e3ea87602f>` , :ref:`VRNA_DECOMP_ML_ML <doxid-group__constraints_1ga7f4cb9ff7a33e67f0539bd39e7b19a78>` , :ref:`VRNA_DECOMP_ML_UP <doxid-group__constraints_1gae6478dda14e50e2f2cb9ef333a29256e>` , :ref:`VRNA_DECOMP_ML_ML_STEM <doxid-group__constraints_1ga63d8ceb8c96ae3b463e529e28cc0fe98>` , :ref:`VRNA_DECOMP_ML_COAXIAL <doxid-group__constraints_1ga4fe48d575830b16c208e280e01ab1497>` , :ref:`VRNA_DECOMP_EXT_EXT <doxid-group__constraints_1ga437adf5115c1999304eff26b41e4c9b6>` , :ref:`VRNA_DECOMP_EXT_UP <doxid-group__constraints_1gaff1ddaffe86d984623910b40cc8a8717>` , :ref:`VRNA_DECOMP_EXT_STEM <doxid-group__constraints_1gae44b5ace0d9b4a29088069ecb4cec441>` , :ref:`VRNA_DECOMP_EXT_EXT_EXT <doxid-group__constraints_1ga803bd818b3f4b2b0a4a5cfa2f7dc2045>` , :ref:`VRNA_DECOMP_EXT_STEM_EXT <doxid-group__constraints_1gabb09c5b78b75a44502fc77b950125c1e>` , :ref:`VRNA_DECOMP_EXT_EXT_STEM <doxid-group__constraints_1ga06efd054c9271438f6d82d4559d9e69f>` , :ref:`VRNA_DECOMP_EXT_EXT_STEM1 <doxid-group__constraints_1ga2e75d7a77118735b32f25422d9686719>` , :ref:`vrna_hc_add_f() <doxid-constraints__hard_8h_1af220427ba7ecc8e786a07b7799658f18>` , :ref:`vrna_hc_add_data() <doxid-constraints__hard_8h_1a128920e0af52e4196a9d59fa13336c7c>`

        void vrna_hc_init (:ref:`vrna_fold_compound_t<doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`* vc)

    .. container:: brief-description

        Initialize/Reset hard constraints to default values.

    .. container:: detailed-description

        This function resets the hard constraints to their default values, i.e. all positions may be unpaired in all contexts, and base pairs are allowed in all contexts, if they resemble canonical pairs. Previously set hard constraints will be removed vefore initialization.
        
        SWIG Wrapper Notes This function is attached as method **hc_init()** to objects of type *fold_compound*
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - vc
        
                - The fold compound

    .. container:: brief-description

        Add hard constraints from pseudo dot-bracket notation.

    .. container:: detailed-description

        This function allows one to apply hard constraints from a pseudo dot-bracket notation. The ``options`` parameter controls, which characters are recognized by the parser.
