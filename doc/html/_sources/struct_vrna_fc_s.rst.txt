Detailed Documentation
----------------------

.. container:: brief-description

    The most basic data structure required by many functions throughout the RNAlib.

.. container:: detailed-description

    SWIG Wrapper Notes This data structure is wrapped as an object **fold_compound** with several related functions attached as methods.
    
    A new **fold_compound** can be obtained by calling one of its constructors:
    
    
    
    * *fold_compound(seq)* Initialize with a single sequence, or two concatenated sequences separated by an ampersand character '&' (for cofolding)
    
    * *fold_compound(aln)* Initialize with a sequence alignment *aln* stored as a list of sequences (with gap characters)
    
    The resulting object has a list of attached methods which in most cases directly correspond to functions that mainly operate on the corresponding *C* data structure:
    
    
    
    * *:ref:`type() <doxid-structvrna__fc__s_1ac5eab693deac9a1a40c2a95ac294707c>`* Get the type of the *fold_compound* (See :ref:`vrna_fc_type_e <doxid-group__fold__compound_1ga01a4ff86fa71deaaa5d1abbd95a1447d>` )
    
    * *:ref:`length() <doxid-structvrna__fc__s_1a95fbfed770b858e50c766505dc4bf998>`* Get the length of the sequence(s) or alignment stored within the *fold_compound*
    
    
    
    .. note::
    
        Please read the documentation of this data structure carefully! Some attributes are only available for specific types this data structure can adopt.
    
    
    
    .. warning::
    
        Reading/Writing from/to attributes that are not within the scope of the current type usually result in undefined behavior!
    
    
    
    .. admonition:: See also:
    
        :ref:`vrna_fold_compound_t.type <doxid-structvrna__fc__s_1ac5eab693deac9a1a40c2a95ac294707c>` , :ref:`vrna_fold_compound() <doxid-group__fold__compound_1ga6601d994ba32b11511b36f68b08403be>` , :ref:`vrna_fold_compound_comparative() <doxid-group__fold__compound_1gad6bacc816af274922b13d947f708aa0c>` , :ref:`vrna_fold_compound_free() <doxid-group__fold__compound_1gadded6039d63f5d6509836e20321534ad>` , :ref:`VRNA_FC_TYPE_SINGLE <doxid-group__fold__compound_1gga01a4ff86fa71deaaa5d1abbd95a1447da7e264dd3cf2dc9b6448caabcb7763cd6>` , :ref:`VRNA_FC_TYPE_COMPARATIVE <doxid-group__fold__compound_1gga01a4ff86fa71deaaa5d1abbd95a1447dab821ce46ea3cf665be97df22a76f5023>`

.. _cid-vrna_fc_s::ptype:
.. container:: fields-container

    .. ref-code-block:: cpp
        :class: title-code-block

        char* ptype

    .. container:: brief-description

        Pair type array.

    .. container:: detailed-description

        Contains the numerical encoding of the pair type for each pair (i,j) used in MFE, Partition function and Evaluation computations.
        
        
        
        .. note::
        
            This array is always indexed via jindx, in contrast to previously different indexing between mfe and pf variants!
        
        
        
        .. warning::
        
            Only available if
        
        .. code-block:: none
        
            type==VRNA_FC_TYPE_SINGLE
        
        
        
        .. admonition:: See also:
        
            :ref:`vrna_idx_col_wise() <doxid-group__utils_1ga89ebc69c52fa0c78c9e1974b0017746b>` , :ref:`vrna_ptypes() <doxid-group__utils_1ga51a9e86a5f731f5f2f5584ee67cee4a8>`
