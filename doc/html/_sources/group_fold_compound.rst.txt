Detailed Documentation
----------------------

.. container:: brief-description

    This module provides interfaces that deal with the most basic data structure used in structure predicting and energy evaluating function of the RNAlib.

.. container:: detailed-description

    Throughout the entire RNAlib, the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` , is used to group information and data that is required for structure prediction and energy evaluation. Here, you'll find interface functions to create, modify, and delete :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` data structures.

    .. container:: brief-description

        Callback to free memory allocated for auxiliary user-provided data.

    .. container:: detailed-description

        This type of user-implemented function usually deletes auxiliary data structures. The user must take care to free all the memory occupied by the data structure passed.
        
        Notes on Callback Functions This callback is supposed to free memory occupied by an auxiliary data structure. It will be called when the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` is erased from memory through a call to :ref:`vrna_fold_compound_free() <doxid-group__fold__compound_1gadded6039d63f5d6509836e20321534ad>` and will be passed the address of memory previously bound to the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` via :ref:`vrna_fold_compound_add_auxdata() <doxid-group__fold__compound_1ga6316a9426bea2f742375e8df6febd3f6>` .

        :ref:`vrna_fold_compound_t<doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`* vrna_fold_compound (
	    const char* sequence,
	    :ref:`vrna_md_t<doxid-group__model__details_1ga1f8a10e12a0a1915f2a4eff0b28ea17c>`* md_p,
	    unsigned int options
	    )

    .. container:: brief-description

        Retrieve a :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` data structure for single sequences and hybridizing sequences.

    .. container:: detailed-description

        This function provides an easy interface to obtain a prefilled :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` by passing a single sequence, or two contatenated sequences as input. For the latter, sequences need to be seperated by an '&' character like this:
        
        .. code-block:: none
        
            char *sequence = "GGGG&CCCC";
        
        The optional parameter ``md_p`` can be used to specify the model details for successive computations based on the content of the generated :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` . Passing NULL will instruct the function to use default model details. The third parameter ``options`` may be used to specify dynamic programming (DP) matrix requirements. Use the macros:
        
        * :ref:`VRNA_OPTION_MFE <doxid-group__fold__compound_1gae63be9127fe7dcc1f9bb14f5bb1064ee>`
        
        * :ref:`VRNA_OPTION_PF <doxid-group__fold__compound_1gabfbadcddda3e74ce7f49035ef8f058f7>`
        
        * :ref:`VRNA_OPTION_WINDOW <doxid-group__fold__compound_1ga2b2a8009ccdccc3eb1571556261aee8e>`
        
        * :ref:`VRNA_OPTION_EVAL_ONLY <doxid-group__fold__compound_1ga61469c423131552c8483229f8b6c7e0e>`
        
        * :ref:`VRNA_OPTION_DEFAULT <doxid-group__fold__compound_1gacea5b7ee6181c485f36e2afa0e9089e4>`
        
        to specify the required type of computations that will be performed with the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` .
        
        If you just need the folding compound serving as a container for your data, you can simply pass :ref:`VRNA_OPTION_DEFAULT <doxid-group__fold__compound_1gacea5b7ee6181c485f36e2afa0e9089e4>` to the ``option`` parameter. This creates a :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` without DP matrices, thus saving memory. Subsequent calls of any structure prediction function will then take care of allocating the memory required for the DP matrices. If you only intend to evaluate structures instead of actually predicting them, you may use the :ref:`VRNA_OPTION_EVAL_ONLY <doxid-group__fold__compound_1ga61469c423131552c8483229f8b6c7e0e>` macro. This will seriously speedup the creation of the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` .
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - sequence
        
                - A single sequence, or two concatenated sequences seperated by an '&' character
        
            *
                - md_p
        
                - An optional set of model details
        
            *
                - options
        
                - The options for DP matrices memory allocation
        
        
        
        .. rubric:: Returns:
        
        A prefilled vrna_fold_compound_t that can be readily used for computations
        
        
        
        .. note::
        
            The sequence string must be uppercase, and should contain only RNA (resp. DNA) alphabet depending on what energy parameter set is used
        
        
        
        .. admonition:: See also:
        
            :ref:`vrna_fold_compound_free() <doxid-group__fold__compound_1gadded6039d63f5d6509836e20321534ad>` , :ref:`vrna_fold_compound_comparative() <doxid-group__fold__compound_1gad6bacc816af274922b13d947f708aa0c>` , :ref:`vrna_md_t <doxid-group__model__details_1ga1f8a10e12a0a1915f2a4eff0b28ea17c>` , :ref:`VRNA_OPTION_MFE <doxid-group__fold__compound_1gae63be9127fe7dcc1f9bb14f5bb1064ee>` , :ref:`VRNA_OPTION_PF <doxid-group__fold__compound_1gabfbadcddda3e74ce7f49035ef8f058f7>` , :ref:`VRNA_OPTION_EVAL_ONLY <doxid-group__fold__compound_1ga61469c423131552c8483229f8b6c7e0e>` , :ref:`VRNA_OPTION_WINDOW <doxid-group__fold__compound_1ga2b2a8009ccdccc3eb1571556261aee8e>`

        void vrna_fold_compound_add_auxdata (
	    :ref:`vrna_fold_compound_t<doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`* vc,

    .. container:: brief-description

        Add auxiliary data to the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` .

    .. container:: detailed-description

        This function allows one to bind arbitrary data to a :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` which may later on be used by one of the callback functions, e.g. :ref:`vrna_callback_recursion_status() <doxid-group__fold__compound_1gac86036fa8cad1108832335063243cdc8>` . To allow for proper cleanup of the memory occupied by this auxiliary data, the user may also provide a pointer to a cleanup function that free's the corresponding memory. This function will be called automatically when the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` is free'd with :ref:`vrna_fold_compound_free() <doxid-group__fold__compound_1gadded6039d63f5d6509836e20321534ad>` .
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - vc
        
                - The fold_compound the arbitrary data pointer should be associated with

    .. container:: brief-description

        Add a recursion status callback to the :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` .

    .. container:: detailed-description

        Binding a recursion status callback function to a :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` allows one to perform arbitrary operations just before, or after an actual recursive computations, e.g. MFE prediction, is performed by the RNAlib. The callback function will be provided with a pointer to its :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` , and a status message. Hence, it has complete access to all variables that incluence the recursive computations.
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - vc
        
                - The fold_compound the callback function should be attached to
        
            *
                - f
        
                - The pointer to the recursion status callback function
