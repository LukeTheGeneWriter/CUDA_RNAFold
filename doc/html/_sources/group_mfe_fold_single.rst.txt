        float vrna_fold (
	    const char* sequence,
	    char* structure
	    )

    .. container:: brief-description

        Compute Minimum Free Energy (MFE), and a corresponding secondary structure for an RNA sequence.

    .. container:: detailed-description

        This simplified interface to :ref:`vrna_mfe() <doxid-group__mfe__fold_1gabd3b147371ccf25c577f88bbbaf159fd>` computes the MFE and, if required, a secondary structure for an RNA sequence using default options. Memory required for dynamic programming (DP) matrices will be allocated and free'd on-the-fly. Hence, after return of this function, the recursively filled matrices are not available any more for any post-processing, e.g. suboptimal backtracking, etc.
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - sequence
        
                - RNA sequence
        
            *
                - structure
        
                - A pointer to the character array where the secondary structure in dot-bracket notation will be written to
        
        
        
        .. rubric:: Returns:
        
        the minimum free energy (MFE) in kcal/mol
        
        
        
        .. note::
        
            In case you want to use the filled DP matrices for any subsequent post-processing step, or you require other conditions than specified by the default model details, use :ref:`vrna_mfe() <doxid-group__mfe__fold_1gabd3b147371ccf25c577f88bbbaf159fd>` , and the data structure :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>` instead.
        
        
        
        .. admonition:: See also:
        
            :ref:`vrna_circfold() <doxid-group__mfe__fold__single_1gaf973483d8acbc8cc9aacfc8a9b7f0074>` , :ref:`vrna_mfe() <doxid-group__mfe__fold_1gabd3b147371ccf25c577f88bbbaf159fd>` , :ref:`vrna_fold_compound() <doxid-group__fold__compound_1ga6601d994ba32b11511b36f68b08403be>` , :ref:`vrna_fold_compound_t <doxid-group__fold__compound_1ga1b0cef17fd40466cef5968eaeeff6166>`

        float vrna_circfold (
	    const char* sequence,
	    char* structure
	    )

    .. container:: brief-description

        Compute Minimum Free Energy (MFE), and a corresponding secondary structure for a circular RNA sequence.

    .. container:: detailed-description

        This simplified interface to :ref:`vrna_mfe() <doxid-group__mfe__fold_1gabd3b147371ccf25c577f88bbbaf159fd>` computes the MFE and, if required, a secondary structure for a circular RNA sequence using default options. Memory required for dynamic programming (DP) matrices will be allocated and free'd on-the-fly. Hence, after return of this function, the recursively filled matrices are not available any more for any post-processing, e.g. suboptimal backtracking, etc.
        
        Folding of circular RNA sequences is handled as a post-processing step of the forward recursions. See :ref:`[7] <doxid-citelist_1CITEREF_hofacker:2006>` for further details.

        .. rubric:: Returns:

        .. note::

        float fold_par (
	    const char* sequence,
	    char* structure,
	    :ref:`vrna_param_t<doxid-group__energy__parameters_1ga8a69ca7d787e4fd6079914f5343a1f35>`* parameters,
	    int is_constrained,
	    int is_circular
	    )

    .. container:: brief-description

        Compute minimum free energy and an appropriate secondary structure of an RNA sequence.

    .. container:: detailed-description

        The first parameter given, the RNA sequence, must be *uppercase* and should only contain an alphabet :math:`\Sigma` that is understood by the RNAlib
        
        (e.g. :math:` \Sigma = \{A,U,C,G\} ` )
        
        The second parameter, *structure* , must always point to an allocated block of memory with a size of at least :math:`\mathrm{strlen}(\mathrm{sequence})+1`
        
        If the third parameter is NULL, global model detail settings are assumed for the folding recursions. Otherwise, the provided parameters are used.
        
        The fourth parameter indicates whether a secondary structure constraint in enhanced dot-bracket notation is passed through the structure parameter or not. If so, the characters " | x < > " are recognized to mark bases that are paired, unpaired, paired upstream, or downstream, respectively. Matching brackets " ( ) " denote base pairs, dots "." are used for unconstrained bases.
        
        To indicate that the RNA sequence is circular and thus has to be post-processed, set the last parameter to non-zero
        
        After a successful call of :ref:`fold_par() <doxid-group__mfe__fold__single_1ga2bc41df5d71fee6fd8da9904ee65d8fb>` , a backtracked secondary structure (in dot-bracket notation) that exhibits the minimum of free energy will be written to the memory *structure* is pointing to. The function returns the minimum of free energy for any fold of the sequence given.
        
        Deprecated use :ref:`vrna_mfe() <doxid-group__mfe__fold_1gabd3b147371ccf25c577f88bbbaf159fd>` instead!
        
        
        
        .. rubric:: Parameters:
        
        .. list-table::
            :widths: 20 80
        
            *
                - sequence
        
                - RNA sequence
        
            *
                - structure
        
                - A pointer to the character array where the secondary structure in dot-bracket notation will be written to
        
            *
                - parameters
        
                - A data structure containing the pre-scaled energy contributions and the model details. (NULL may be passed, see OpenMP notes above)
        
            *
                - is_constrained
        
                - Switch to indicate that a structure constraint is passed via the structure argument (0==off)
        
            *
                - is_circular
        
                - Switch to (de-)activate post-processing steps in case RNA sequence is circular (0==off)
        
        
        
        .. rubric:: Returns:
        
        the minimum free energy (MFE) in kcal/mol
        
        
        
        .. note::
        
            OpenMP: Passing NULL to the 'parameters' argument involves access to several global model detail variables and thus is not to be considered threadsafe
        
        
        
        .. admonition:: See also:
        
            :ref:`vrna_mfe() <doxid-group__mfe__fold_1gabd3b147371ccf25c577f88bbbaf159fd>` , :ref:`fold() <doxid-group__mfe__fold__single_1gaadafcb0f140795ae62e5ca027e335a9b>` , :ref:`circfold() <doxid-group__mfe__fold__single_1ga4ac63ab3e8d9a80ced28b8052d94e423>` , :ref:`vrna_md_t <doxid-group__model__details_1ga1f8a10e12a0a1915f2a4eff0b28ea17c>` , set_energy_model(), :ref:`get_scaled_parameters() <doxid-group__energy__parameters_1ga7fa6a000d7c16feab939f2c4ee626197>`
